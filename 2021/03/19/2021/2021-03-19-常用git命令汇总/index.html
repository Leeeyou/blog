常用git命令汇总

Posted on 2021-03-19 Edited on 2021-03-22 In code

0. 基础linux命令

0.1 清除屏幕

clear

往控制台输出信息

echo ‘test content’
echo ‘test content’ > test.txt

查看当前处在哪个目录下

pwd

将当前目录下的自文件&子目录平铺在控制台

ls
ls -ll

将对应目录下的子孙文件&子孙目录平铺在控制台

find [目录名]

将对应目录下的文件平铺在控制台

find [目录名] -typef

删除文件

rm [文件名]

重命名

mv [源文件] [重命名文件]

查看对应文件的内容

cat [文件的url]

编辑文件

vim [文件的url]

  • 按i键进入插入模式
  • 按esc键进行命令执行
    • q! 强制退出(不保存)
    • wq 保存退出
    • set nu 设置行号

1. 新建仓库

1.1 在当前目录新建一个git代码库

git init

1.2 新建一个目录，将其初始化为git代码库

git init [project-name]

1.3 下载一个项目和它到整个代码历史

git clone [url]

2. 配置

2.1 显示当前的git配置

git config –list

2.2 编辑git配置文件

git config -e [–global]

2.3 设置提交代码时的用户信息

git config [–global] user.name “[name]”
git config [–global] user.email “[email address]”

3. 增加/删除文件

3.1 添加当前目录的所有文件到暂存区

git add .

3.2 删除工作区文件，并且将这次删除放入暂存区

git rm [file1] [file2] …

3.3 停止追踪指定文件，但该文件会保留在工作区

git rm –cached [file1]

3.4 改名文件，并且将这个改名放入暂存区

git mv [file-original] [file-renamed]

4. 代码提交

4.1 提交暂存区到仓库区

git commit -m [message]

4.2 将已commit过的文件直接提交到仓库区

git commit -a -m [message]

4.3 提交时显示所有diff信息

git commit -v

4.4 使用一次新的commit替代上一次提交，如果代码没任何变化，则用来改写上一次commit的提交信息

git commit -amend
git commit -amend -m [message]

4.5 重做上一次commit，并包括指定文件的新变化

git commit -amend [file1] [file2] …

5. 分支

5.1 列出所有本地分支

git branch

5.2 列出所有远程分支

git branch -r

5.3 列出所有本地分支和远程分支

git branch -a

5.4 新建一个分支，但依然停留在当前分支

git branch [branch-name]

5.5 新建一个分支，并切换到该分支

git checkout [branch] [commit]

5.6 切换到指定分支，并更新工作区

git checkout [branch-name]

5.7 切换到上一个分支

git checkout -

5.8 合并指定分支到当前分支

git merge [branch]

5.9 选择一个commit合并进当前分支

git cherry-pick [commit]

5.10 删除分支

git branch -d [branch-name]

5.11 强制删除分支，忽略该分支的变化

git branch -D [branch-name]

5.12 删除远程分支

git push origin –delete [branch-name]
git branch -dr [remote/branch]

6. 标签

6.1 列出所有tag

git tag

6.2 在当前commit新建一个tag

git tag [tag]

6.3 在指定commit新建一个tag

git tag [tag] [commit]

6.4 删除本地tag

git tag -d [tag]

6.5 删除远程tag

git push origin :refs/tags/[tagName]

6.6 查看tag信息

git show [tag]

6.7 提交指定tag

git push [remote] [tag]

6.8 提交所有tag

git push [remote] –tags

6.9 新建一个分支，指向某个tag

git checkout -b [branch] [tag]

7. 查看信息

7.1 显示有变更的文件

git status

7.2 显示当前分支的历史版本

git log

7.3 显示commit历史以及每次commit发生变更的文件

git log –stat

7.4 根据关键词搜索提交历史

git log -S [keyword]

7.5 显示过去5次提交

git log -5 –oneline

7.6 显示所有提交过的用户，按提交次数排序

git shortlog -sn

7.7 显示指定文件是什么人在什么时间修改过

git blame [file]

7.8 显示暂存区和工作区的差异

git diff

7.9 显示暂存区和上一个commit的差异

git diff –cached [file]

7.10 显示工作区与当前分支最新commit之间的差异

git diff HEAD

7.11 显示两次提交之间的差异

git diff [first-branch]…[second-branch]

7.12 显示已被修改的文件已被暂存但还未提交

git diff –staged

7.13 显示当前分支的最近几次提交

git reflog

7.14 查看暂存区内容

git ls-files -s

8. 远程同步

8.1 下载远程仓库的所有变动

git fetch [remote]

8.2 显示所有远程仓库

git remote -v

8.3 显示某个远程仓库的信息

git remote show [remote]

8.4 增加一个新的远程仓库并命名

git remote add [shortname] [url]

8.5 取回远程仓库的所有变化并与本地分支合并

git pull [remote] [branch]

8.6 上传本地指定分支到远程仓库

git push [remote] [branch]

8.7 强行推送当前分支到远程仓库，即使有冲突

git push [remote] –force

8.8 推送所有分支到远程仓库

git push [remote] –all

9. 撤销

9.1 恢复暂存区的指定文件到工作区

git checkout [file]

9.2 恢复某个commit的指定文件到暂存区和工作区

git checkout [commit] [file]

9.3 恢复暂存区的所有文件到工作区

git checkout .

9.4 重置暂存区的指定文件，与上一次commit保持一致，但工作区不变

git reset [file]

9.5 重置暂存区与工作区，与上一次commit保持一致

git reset –hard

9.6 重置当前分支的HEAD为指定commit，同时重置暂存区和工作区，与指定commit一致

git reset –hard [commit]

9.7 重置当前HEAD为指定commit，但保持暂存区和工作区不变

git reset –keep [commit]

9.8 新建一个commit，用来撤销指定commit，后者的所有变化都将被前者抵消，并且应用到当前分支

git revert [commit]

10. 存储

10.1 将未完成的修改保存到一个栈上

git stash

10.2 应用上一次的存储

git stash apply

10.3 查看存储

git stash list

10.4 应用指定的存储

git stash apply [stash@{2}]

10.5 移除指定的存储

git stash drop [stash@{0}]

10.6 应用指定的存储然后将其出栈

git stash pop [stash@{0}]

11. 其他

11.1 生成一个可供发布的压缩包

git archive
